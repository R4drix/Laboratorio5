2. Redes y protocolos básicos

Las redes conectan dispositivos para compartir datos. Para que esa comunicación funcione existen protocolos —conjuntos de reglas— que definen cómo se empaquetan, envían y reciben los datos. Ejemplos conocidos son TCP/IP para transmisión fiable y HTTP/HTTPS para la web. Conocer el modelo en capas (por ejemplo, el modelo OSI) ayuda a entender dónde actúa cada protocolo y cómo se puede diagnosticar un problema de conectividad.

Referencias: Introducción a redes (MDN) · Protocolo de red (Wikipedia)

1. Introducción a los algoritmos

Un algoritmo es una serie de pasos ordenados y finitos para resolver un problema o realizar una tarea. En computación, diseñar un algoritmo claro permite que una máquina (o una persona) ejecute tareas de forma eficiente y predecible. Los criterios habituales para evaluar algoritmos son la corrección (resuelven el problema) y la complejidad (cuánto tiempo y memoria requieren). Aprender a describir algoritmos con pseudocódigo o diagramas de flujo ayuda a pensar de forma estructurada antes de pasar a programar.

Referencias: Algoritmo (Wikipedia) · ¿Qué es un programa? (MDN)

3. Seguridad básica en aplicaciones web

La seguridad en aplicaciones web comienza por validar correctamente las entradas de usuario, usar conexiones cifradas (HTTPS) y manejar sesiones de forma segura. Ataques comunes incluyen inyección SQL, Cross-Site Scripting (XSS) y Cross-Site Request Forgery (CSRF). Implementar controles como saneamiento de datos, cabeceras de seguridad y autenticación robusta reduce riesgos y protege tanto a los usuarios como a los datos del sistema.

Referencias: OWASP Top 10 · Seguridad web (MDN)
